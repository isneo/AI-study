Memory 专题拆解避免只讲“Agent 有记忆”这种泛泛概念能讲清楚风控场景中 memory 的价值、边界、风险和验证方式按长期记忆、短期上下文、业务事实、用户偏好、案例经验、SOP 技能分别整理：存什么、谁能写、何时更新、如何回滚、如何验证收益

落地选型矩阵区分直接集成、思路借鉴和自研，避免框架名堆砌面试和方案评审时能解释为什么选、为什么不选对 LangGraph、Spring AI、OpenAI/Anthropic SDK、Pi/DeerFlow、MCP/A2A、Langfuse 等按层级、适用场景、风险、集成成本做一张矩阵

系统演进复盘回答“为什么一开始不是 agentic workflow，如果重做会怎样”形成更可信的工程判断，而不是事后套概念梳理传统 workflow 的合理性、触发 agentic workflow 的条件、迁移路径、风险控制和阶段性收益
资料短摘要补齐现在资料多但入口仍偏重文件名让后来者不用逐个打开大文件也能知道先看什么给 HTML、Markdown、PDF 各补 3-5 句摘要、适合人群、阅读顺序，更新到 resource-catalog.md
图片资料二次补充当前图片只能拿到不可读缓存，缺少原始截图类材料让图示、截图和讨论上下文更完整从原始客户端手动另存确认为学习资料的图片，放入 assets/images/，避免导入生活照、聊天截图和隐私图片
