学习主题索引

1. 横向认知到风控应用

核心问题：业界 Agent 能力如何迁移到风控业务，而不是停留在概念复述。

可复习要点：

  • 泛化能力：哪些 Agent 能力是跨业务可复用的，例如工具调用、trace、任务分解。
  • 延展性：如何从单点任务扩展到流程级自动化。
  • 风控优势：B 端场景更容易拿到 Ground Truth，更适合做数据闭环和效果验证。

2. Agent 业界调研和架构选型

核心问题：不同框架和 SDK 解决的问题不同，不能直接横向混比。

可复习要点：

  • LangGraph：偏底层 orchestration runtime。
  • Pi / DeerFlow：偏完整 agent harness。
  • OpenAI / Anthropic SDK：偏模型厂商提供的 runner、tool、handoff、trace SDK。
  • Spring AI 等框架：可作为工程集成层参考。

选型时优先回答：

  • 它解决 runtime、harness、tool、memory、trace 中哪一层。
  • 是直接集成、借鉴思想，还是需要自研。
  • 它和风控系统边界是否一致。

3. Agent 三层表达框架

群内讨论倾向用更容易记忆的结构表达：

  • 上下文与记忆：决定模型知道什么、保留什么。
  • 约束与验证：决定模型能不能被业务系统接受。
  • 运行时与编排：决定任务怎么执行、如何流转、如何观测。

这个框架适合用于面试、复盘和方案汇报。

4. 差异化故事

核心问题：不只是说「用了 Agent」，而是说明为什么跑得快、怎么跑、和行业有什么不同。

可复习要点：

  • 跑得快：是否有业务数据闭环、工程底座、评估体系。
  • 怎么跑：是否有从 workflow 到 agentic workflow 的清晰演进路径。
  • 为什么不同：是否结合了风控场景的 Ground Truth、约束验证和业务专家经验。

5. 后续独立专题

  • Memory 机制：技术和业务一起讨论。
  • 落地方案选型：集成、借鉴、自研的边界。
  • 系统演进复盘：如果重做，哪些地方会从一开始引入 agentic workflow。
