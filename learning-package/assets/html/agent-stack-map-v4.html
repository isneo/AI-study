Agent Stack Map v4 · Five-layer Stable Model

Agent 技术栈全景图：五层稳定版

v4 收敛为五层：产品层、Runtime/Harness 层、App Framework 层、Capability Components 层、Model/Data/Tool Infrastructure 层。原 L4/L5 合并成 Runtime/Harness，并在层内拆 Orchestration、Execution、Memory/State、Governance/Observability 四个平面；原 L1/L2 合并，去掉通用云基础设施噪音。

核对日期：2026-05-23
定位：Java 后端转 AI Agent 工程

v4 分层原则

主图只保留 Agent 特异能力。K8s、Redis、IAM、CI/CD 这类通用后端基础设施不单独成层，只在生产化讨论中作为旁注。

核心判断

  • LangGraph、DeerFlow、OpenHands、Pi 都属于 L4 Runtime/Harness 大层，但侧重点不同。
  • Spring AI、LangChain、LlamaIndex 属于 L3 App Framework / SDK。
  • Coze、Dify、Claude Code、Codex 属于 L5 产品/平台层。
  • MCP、VectorIndex、Guardrails、Eval 属于 L2 Agent 能力组件。

面试总答法

“我会把 Agent 技术栈拆成五层：底层是模型、数据和工具基础；再上面是 MCP、向量检索、评测、Guardrails 这些 Agent 能力组件；再上面是 Spring AI、LangChain、LlamaIndex 这样的开发框架；核心是 Agent Runtime/Harness，里面分编排平面和执行平面；最上面是 Coze、Dify、Claude Code、Codex 或企业业务 Agent 这种产品层。”
