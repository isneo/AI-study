推荐路线

如果目标是“手机上随时管理 Claude/Codex，并逐步进化成个人助理”，不要一开始就把所有平台都装上。先把入口跑稳，再把记忆和数据接进来。

1

先把入口稳定下来

用 Dinotty 做手机 Web 终端，Tailscale 提供安全内网访问。这个组合最符合你已经在 Mac 和三星手机上跑通的状态。

2

再补上下文记忆

引入 agentmemory 或 OpenHuman 的记忆树，让 Codex、Claude、pi 等工具共享长期背景，减少每次重新解释项目和偏好的成本。

3

最后考虑平台化

OpenHuman、GenericAgent、cc-connect 属于更大的个人助手/自动化平台。它们潜力大，但配置面和权限面也更复杂，适合在入口稳定后逐步试。

怎么取舍

这些项目不是互斥关系。它们更像不同层：网络层、入口层、Agent 层、记忆层、路由层。

适合你立刻用的组合

  • Dinotty 负责手机终端入口，Tailscale 负责安全联网。
  • Codex/Claude 继续跑在 Mac 上，手机只是控制和查看。
  • 如果要看成本，再把 9router 或 Sub2API 放到 API 网关层。

不建议现在一口气全上

  • OpenHuman 和 GenericAgent 都会触达大量本地数据和账号权限，需要慢慢验证边界。
  • Happy、cc-connect、Dinotty 都是“远程控制 Coding Agent”的入口形态，先选一个主入口。
  • 9router 和 Sub2API 都涉及模型请求转发，要单独评估账号风险、服务稳定性和合规边界。
